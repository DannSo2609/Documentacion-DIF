Deshabilitar/Apagar un entorno administrado por Power Platform

https://learn.microsoft.com/es-es/power-platform/admin/managed-environment-enable
¿Qué debe hacer si habilitó accidentalmente un entorno administrado en Power Platform y desea que vuelva a estar como no administrado? No es posible deshabilitar o desactivar un entorno administrado en el Centro de administración de Power Platform; solo puede habilitarlo o eliminarlo en el Centro de administración. Para deshabilitar o desactivar un entorno administrado y volver a un entorno no administrado, debe usar PowerShell. A continuación, le explicamos cómo hacerlo:
 
 
 
 
Abra un símbolo del sistema de PowerShell como administrador.
1-Instalar PowerShell
Install-Module -Name Microsoft.PowerApps.Administration.PowerShell
2-Inicie sesión con una cuenta con permisos para administrar entornos. El siguiente comando abrirá un mensaje de autenticación. Este mensaje admite MFA.
Add-PowerAppsAccount -Endpoint prod
Ejecute este código que establece protectionLevel en Básico, lo que convierte un entorno de administrado a no administrado.
$UpdatedGovernanceConfiguration = [pscustomobject]@{
protectionLevel = "Basic"
}
Ejecute este comando para aplicar los cambios al entorno que desea convertir de administrado a no administrado. Asegúrese de reemplazar el ID del entorno según corresponda.
Set-AdminPowerAppEnvironmentGovernanceConfiguration -EnvironmentName <PUT THE ENVIRONMENT ID HERE> -UpdatedGovernanceConfiguration $UpdatedGovernanceConfiguration

Deshabilitar entornos administrados mediante PowerShell
Los administradores pueden usar PowerShell para eliminar la propiedad Entornos administrados de un entorno. Antes de deshabilitarlos, el administrador debe asegurarse de que ninguna de sus funciones esté en uso.
A continuación se muestra un ejemplo de script de PowerShell que llama a la API para establecer la propiedad Entornos administrados:
PowerShell
$UpdatedGovernanceConfiguration = [pscustomobject]@{
    protectionLevel = "Basic"
}
Set-AdminPowerAppEnvironmentGovernanceConfiguration -EnvironmentName 8fb9b601-7b3a-e294-be2a-936826884211 -UpdatedGovernanceConfiguration $UpdatedGovernanceConfiguration

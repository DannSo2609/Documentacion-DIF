Archivo de mapeo de usuarios manualmente

Siga los pasos a continuación para preparar el archivo de mapeo de usuarios: 1. Inicie sesión en el Centro de administración de Power Platform (PPAC) y abra el entorno.

1.     Click on Access -> Users -> See all.
2.     Click on Manage users in Dynamics 365.
3.     Select Edit columns.
4.     Select Add columns.
5.     Add only the Primary Email column and remove all other existing columns.
6.     Select Apply.
7.     Click Export to Excel.
8.     Create a new CSV file using Excel.
9.     Create two columns: Source and Destination.
10.  Paste the Primary email address exported in Step 8 under the Source column.
11.  Provide the mapping email address in the destination tenant under the Destination column.
12.  Save the file as usermapping.csv (case-sensitive).

1. View migration Request
  • View the migration request using the TenantToTenant-ViewMigrationRequest cmdlet.
    2. Approve Migration Request:
  • Approve the request using the TenantToTenant-ManageMigrationRequest cmdlet.
    3. Generate SAS URL:
  • Generate the SAS URL for resource storage.
    4. Upload User Mapping File:
  • Upload the user mapping file to Azure Blob Storage.
    5. Get Migration Status:
  • Check the status using the TenantToTenant-GetStatus cmdlet.
    6. Prepare for Migration:
  • Prepare the environment using the TenantToTenant-PrepareMigration cmdlet.
    7. Migrate the Environment:
  • Execute the migration using the TenantToTenant-MigratePowerAppEnvironment cmdlet.
 
Nota: Comando importante para la migración
• El administrador del paso 5 debe ejecutar el comando "Ver migración" (let) con su propio ID de inquilino. • Todos los pasos deben ser realizados por el administrador de origen, excepto los pasos 5 y 6, que deben ser realizados por los usuarios administradores del inquilino de destino. • Después del paso 9, asegúrese de ejecutar la función "Obtener estado". Si hay algún problema con el archivo de mapeo de usuarios, se le proporcionará la URL de SAS para descargar el informe de mapeo de usuarios fallido. Deberá corregirlo y volver a subirlo. Para comenzar desde el paso 8, utilice la misma URL de SAS o genere una nueva. • Después del paso 9, si la función "Obtener estado" se ha realizado correctamente, solo entonces puede omitir el paso 11 y continuar con el 12.Use the following PowerShell script step -by step sequentially

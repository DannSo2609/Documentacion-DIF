Step 2. Migration process

Ensure the following steps are completed sequentially to move environment between tenants with appropriate parameters.
1. Install PowerShell Modules: (Learn more)
  • Install PowerShell for Power Platform administrators.
  • Install Azure PowerShell on Windows.
  • Add-PowerAppsAccount -Endpoint prod
 
 
Install-Module -Name Az -Repository PSGallery -Force
 
2. Submit Migration Request:
  • Use the TenantToTenant-SubmitMigrationRequest cmdlet with the appropriate parameters.
 
3. View migration Request
  • View the migration request using the TenantToTenant-ViewMigrationRequest cmdlet.
    4. Approve Migration Request:
  • Approve the request using the TenantToTenant-ManageMigrationRequest cmdlet.
    5. Generate SAS URL:
  • Generate the SAS URL for resource storage.
    6. Upload User Mapping File:
  • Upload the user mapping file to Azure Blob Storage.
    7. Get Migration Status:
  • Check the status using the TenantToTenant-GetStatus cmdlet.
    8. Prepare for Migration:
  • Prepare the environment using the TenantToTenant-PrepareMigration cmdlet.
    9. Migrate the Environment:
  • Execute the migration using the TenantToTenant-MigratePowerAppEnvironment cmdlet.
Notes:
·         Step 5 admin should execute View migration command let by passing their own tenantID.
·         All the steps need to be performed by source admin except step 5 & 6 that needs to be performed by target tenant admin users.
·         After step 9 make sure to run the get status and any issue with user mapping file you will be provided with SAS url to download failed user mapping report that you will need to fix and re-upload and start from step 8 using same SAS URL or you can generate new SAS URL too.
·         After step 9 if get status is succeeded then only you can skip step 11 and procced with 12

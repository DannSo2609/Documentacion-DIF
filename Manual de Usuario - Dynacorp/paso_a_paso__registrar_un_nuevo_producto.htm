Paso a Paso: Registrar un Nuevo Producto

🔄 ¿Qué pasa después?
  • El país queda disponible para ser usado en:
  • Fichas de clientes
  • Embarques (origen/destino)
  • Declaraciones SIGA
  • Facturación internacional
  • Reportes por zona o continente
📝 Validaciones automáticas
  • El sistema evitará duplicar países con el mismo código ISO.
  • Si un país está inactivo, no podrá ser asignado en nuevas operaciones.
  • Países preconfigurados por integración (ej. con SIGA o Magaya) no deben eliminarse.
🟡 Notas importantes
  • Todos los países utilizados en operaciones deben tener códigos ISO válidos para exportaciones y sistemas aduanales.
  • No se deben registrar regiones o ciudades como países (ej. "Miami", "Europa").
  • Verifica el uso del país antes de desactivarlo: puede afectar embarques abiertos o cotizaciones en curso.
✅ Buenas prácticas
  • Revisa que todos los países usados estén normalizados con sus códigos ISO actualizados.
  • Si el sistema se integra con SIGA, asegúrate de usar los mismos códigos homologados.
  • Si manejas zonas o regiones personalizadas, usa otro catálogo (Ej. Zonas logísticas, no países).
